# Cloud Agent Platform — 项目架构文档

> **版本**: 0.1.0 | **Python**: >= 3.14 | **最后更新**: 2025-07

---

## 1. 项目概述

Cloud Agent Platform（CAP）是一个基于 Python 的 **AI Agent 工具框架**，对标 Claude Code 工具系统设计。它提供交互式命令行对话代理，支持 LLM function calling 模式下的工具编排、会话持久化与恢复。

核心能力：

- **交互式 REPL**：通过 CLI 启动对话代理，持续接收用户输入
- **LLM + 工具编排**：外层用户轮次循环 + 内层 LLM tool_calls 循环，支持连续多步工具调用
- **会话持久化**：基于 JSON 文件的会话存储，支持会话的创建/列表/恢复/删除
- **可扩展工具系统**：16 个内置工具，统一继承 `BaseTool` 并通过 `ToolRegistry` 管理
- **流式响应**：支持流式 LLM 输出及 extended thinking（推理链）

## 4. 核心模块详解

### 4.1 CLI 入口 (`main.py`)

**职责**: 命令行参数解析 + 环境校验 + 命令分发。

| 命令 | 功能 |
|------|------|
| `python main.py create` | 创建新会话并启动交互式代理 |
| `python main.py list` | 列出所有已保存的会话摘要 |
| `python main.py resume --id <ID>` | 恢复已有会话继续对话 |
| `python main.py delete --id <ID>` | 删除指定会话 |
| `python main.py version` | 显示版本和 Python 版本 |
| `python main.py` (无参数) | 默认启动 create 流程 |

**环境变量配置** (`.env`):
- `LLM_MODEL_ID` — 模型 ID（如 `deepseek-v4-pro`）
- `LLM_API_KEY` — API 密钥
- `LLM_BASE_URL` — API 服务地址
- `LLM_TIMEOUT` — 超时秒数（默认 60）

---

### 4.2 Agent 层 (`agent/agent.py`)

**Agent 类** 是对话代理的核心引擎，采用 **双层循环** 架构：

```
外层 while 循环（用户轮次）
  ├─ 接收用户输入 (input)
  ├─ 将用户消息追加到 self.messages
  ├─ 调用 self._run_turn()
  └─ turn_count += 1

内层 for 循环（LLM 轮次，上限 INNER_LOOP_LIMIT=20）
  ├─ self.llm.think(messages, tools, effort)
  ├─ finish_reason == "stop"    → 打印回复，退出内循环
  ├─ finish_reason == "error"   → 打印错误，退出
  ├─ finish_reason == "length"  → 输出截断警告，退出
  └─ tool_calls 存在           → 逐条执行工具，结果追加到 messages，继续内循环
```

**关键设计**:
- `self.messages` 归属 Agent，跨越内外循环保留完整历史
- System prompt 由 `_init_system_prompt()` 根据已注册工具动态生成
- 每个工具执行结果序列化为 JSON 字符串，与 OpenAI tool role 消息格式兼容

---

### 4.3 LLM API 层 (`llmapi/LLMAPI.py`)

**LLMAPI 类** 封装 OpenAI 兼容的 LLM 调用：

- 基于 `openai` SDK 的 `chat.completions.create()`
- 默认使用 **流式响应** (`stream=True`)
- 支持 **extended thinking**（通过 `extra_body.thinking` 参数控制推理链输出）
- 流式累积：文本内容、推理链内容、tool_calls（按 index 聚合 Delta）
- 返回结构化 dict：
  ```python
  {
      "finish_reason": str,          # "stop" | "tool_calls" | "error" | "length"
      "content": Optional[str],      # 文本回复
      "reasoning_content": Optional[str],  # 推理链
      "tool_calls": Optional[List[dict]],  # 工具调用
      "error": Optional[str],        # 仅 finish_reason="error" 时
  }
  ```
- 自动推断 finish_reason：部分 API 不发送 finish_reason 但 tool_calls 已完整，此时推断为 `"tool_calls"`

### 4.4 工具系统 (`tools/`)

#### 4.4.1 BaseTool（抽象基类）

```python
class BaseTool(ABC):
    name: str = ""
    description: str = ""

    async def execute(self, **kwargs) -> ToolResult: ...   # abstract
    def get_schema(self) -> Dict[str, Any]: ...            # abstract
    def validate_params(self, params: Dict[str, Any]): ...  # no-op
```

#### 4.4.2 ToolResult（返回结果）

```python
@dataclass
class ToolResult:
    success: bool
    data: Optional[Any] = None
    error: Optional[str] = None
    metadata: Dict[str, Any] = {}
```

#### 4.4.3 ToolRegistry（注册表）

| 方法 | 说明 |
|------|------|
| `register(tool)` | 注册工具（校验类型 + 名称非空） |
| `unregister(name)` | 注销工具 |
| `get(name)` → `Optional[BaseTool]` | 按名称查找 |
| `list_tools()` → `List[dict]` | 所有工具的 `{name, description}` |
| `get_all_schemas()` → `List[dict]` | 所有工具的 JSON Schema（供 LLM function calling） |
| `clear()` | 清空注册表 |

### 4.5 会话管理 (`session/`)

#### 4.5.1 Message 模型

```python
class Message(BaseModel):
    role: str               # "system" | "user" | "assistant" | "tool"
    content: Optional[str]
    tool_calls: Optional[List[Dict[str, Any]]]
    tool_call_id: Optional[str]
    name: Optional[str]
    reasoning_content: Optional[str]
```

#### 4.5.2 Session 模型

```python
class Session(BaseModel):
    id: str                 # uuid4().hex[:12]
    name: str
    agent_name: str
    messages: List[Message]
    turn_count: int
    max_turns: int
    created_at: str         # ISO 时间戳
    updated_at: str
```

关键方法：
- `add_message(msg)` — 添加消息并刷新 `updated_at`
- `to_api_messages()` — 转换为 LLM API 格式（跳过 system 消息）
- `get_system_prompt()` — 获取系统提示词

#### 4.5.3 SessionManager

| 方法 | 说明 |
|------|------|
| `create(name, agent_name, max_turns)` → `Session` | 创建新会话并立即持久化 |
| `save(session)` | 序列化到 `memory/sessions/<id>.json` |
| `load(session_id)` → `Optional[Session]` | 从磁盘加载 |
| `list_sessions()` → `List[dict]` | 列出所有会话摘要（按更新时间倒序） |
| `delete(session_id)` → `bool` | 删除会话文件 |
| `session_exists(session_id)` → `bool` | 检查会话是否存在 |

存储格式：`memory/sessions/<safe_id>.json`（每个会话一个 JSON 文件，ID 经过安全过滤仅保留字母数字和连字符）

## 5. 数据流

### 5.1 典型一轮对话流程

```
用户输入 "读取 main.py"
        │
        ▼
Agent.messages.append({"role": "user", "content": "读取 main.py"})
        │
        ▼
Agent._run_turn()
  ├─ llm.think(messages, tools=[...schemas...])
  │     │
  │     ▼ OpenAI API 流式响应 → 聚合为结构化 dict
  │     finish_reason = "tool_calls"
  │     tool_calls = [{"function": {"name": "read_file", "arguments": "{...}"}}]
  │
  ├─ assistant_msg → messages
  │
  ├─ _execute_tool("read_file", {"file_path": "main.py"})
  │     │
  │     ▼ ReadFileTool.execute() → ToolResult(success=True, data={...})
  │     序列化为 JSON → messages.append({"role": "tool", ...})
  │
  ├─ llm.think(messages, tools=[...])  # 第二次调用，消化工具结果
  │     │
  │     ▼ finish_reason = "stop"
  │     content = "main.py 包含以下内容：..."
  │
  └─ messages.append({"role": "assistant", "content": "..."})
        │
        ▼
打印 "main.py 包含以下内容：..."
```

### 5.2 会话恢复流程

```
python main.py resume --id abc123def456
        │
        ▼
SessionManager.load("abc123def456") → Session 对象
        │
        ▼
Session.to_api_messages() → List[dict]  (不含 system 消息)
        │
        ▼
Agent.messages = [system_prompt] + restored_messages
Agent.turn_count = session.turn_count
        │
        ▼
Agent.run() → 继续交互
```

## 7. 扩展点

1. **新工具接入**: 继承 `BaseTool`，实现 `execute()` 和 `get_schema()`，在 `create_default_registry()` 中注册即可
2. **换模型**: 修改 `.env` 中的 `LLM_MODEL_ID` / `LLM_BASE_URL` / `LLM_API_KEY`，支持任意 OpenAI 兼容接口
3. **会话后端**: 目前为 JSON 文件存储，可扩展为 SQLite / PostgreSQL 通过替换 `SessionManager`
4. **骨架工具完善**: `web_search`、`start_agent`、`task_create` 等 8 个工具目前仅有 Schema 定义，需接入实际后端
5. **子代理 Runtime**: `start_agent` 和 `run_workflow` 需要上层 Agent Runtime 支持子代理生命周期管理
